# 任务描述

我需要在devproc2中支持openpi0.5模型的编译运行。
openpi0.5模型的repo在/root/autodl-tmp/openpi。该repo目录下有uv环境，使用改python环境，运行/root/autodl-tmp/openpi/scripts/check_pi05_fp16_infer.py。即可跑通pi0.5模型的推理。

在devproc2中支持该模型的推理，需要完成：
1. devproc2 dsl前端支持类似 torch nn.Module的写法，来搭建pi0.5的模型结构。
2. 支持所需的nn module class，比如Embedding, Linear, LayerNorm, GELU等。
3. nn.module 底层forward 使用devproc2 ir支持的op。op 是存在attr的，目前devproc2还没有支持attr系统。需要设计op的attr。
4. 为每个op实现triton kernel。实现lowering。
5. 完成weight的映射，将huggingface 格式的weight映射到devproc2 ir中的weight。要设计优雅的weight映射机制，此外，还要考虑后续如何支持量化。量化一般是在外部框架，比如torchao等中完成，完成后，将量化后的weight映射到devproc2 ir中。本次任务不需要实现量化，但需要考虑后续如何实现，为后续实现提供基础建设。
6. 编译。openpi0.5模型在devproc2中搭建好之后，需要能够进行编译，生成编译产物。编译时需要将所有cuda kernel编译出cubin。
7. 运行。需要在c++ 侧实现纯c++的运行，不依赖python。python侧只需要调用c++的运行接口即可。
8. 精度对齐。对齐devproc2 openpi0.5模型的精度，与pytorch的精度一致。

本次任务不做任何代码实现，而是根据当前devproc2的现状，梳理一下，设计完整的支持方案。方案要分阶段实现，列出milestone。
