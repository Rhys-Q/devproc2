# 任务描述

当前pi0.5 模型在devproc2中的实现还有很多不符合设计的地方。
pi05实现在ython/devproc2/models/pi05。

## 1 nn.Module实现不规范
在设计规范中，nn.Module对外只有forward函数和forward_fast函数。forward 函数提供标准实现，forward_fast函数提供hpc高性能实现。
当前很多nn.Module不符合这个设计规范，需要改造。

## 2 cuda kernel 接入实现丑陋
当前在构图时，通过call_pi05_cuda_kernel函数做了一层warp。
预期不应该有这一层warp。应该直接：
``` python
dp.cuda_call(
            "kernels/add_one.cu::add_one",
            x,
            y,
            4,
            metadata={"grid": (1, 1, 1), "block": (64, 1, 1)},
        )
```
